# Clase 18 - Kotlin

## Introducción

- Lenguaje de programación **orientado a objetos** de **tipado estático** que corre sobre la **máquina virtual de Java (JVM)** y también puede compilarse a **JavaScript**.
- Desarrollado inicialmente por **JetBrains**.
- Código fuente bajo **Licencia Apache 2**.
- Publicado por primera vez en **2011**.
- Ejecuta en cualquier SO con soporte JVM o un intérprete de JavaScript.

## Beneficios

- **Expresivo y conciso**: reduce la cantidad de código → mayor productividad.
- **Código más seguro**: funciones que evitan errores comunes como las **excepciones de puntero nulo (null pointer)**. Estadísticamente, las apps en Kotlin tienen un **20% menos de probabilidad de fallar**.
- **Interoperabilidad**: permite llamar código Java y viceversa. **100% interoperable con Java** → adopción progresiva.
- **Concurrencia estructurada**: facilita el código asíncrono. Las **corrutinas (coroutines)** simplifican la administración de tareas en segundo plano (llamadas de red, acceso a datos locales / base de datos).

## Temas cubiertos

- Playground (entorno online para probar código).
- Declaración de variables.
- Condicionales.
- Funciones.
- Clases.
- Recursos: Android Basics with Kotlin, documentación oficial (`kotlinlang.org/docs/basic-syntax`), Kotlin Bootcamp (Udacity).

## Preguntas típicas V/F

- Kotlin es un lenguaje de tipado dinámico → **Falso** (tipado **estático**).
- Kotlin corre sobre la JVM y es 100% interoperable con Java → **Verdadero**.
- Kotlin fue desarrollado por JetBrains → **Verdadero**.
- Las corrutinas de Kotlin sirven para manejar tareas asíncronas / en segundo plano → **Verdadero**.
- Kotlin ayuda a evitar excepciones de puntero nulo → **Verdadero**.
- Kotlin solo puede compilarse a bytecode de la JVM → **Falso** (también a JavaScript).
